# YTMetrics — настольное приложение (Electron)

Это обёртка над вашим уже работающим сайтом (`main.js` просто открывает
`https://jakjuk523.github.io/youtube-analytics` в нативном окне). Это НЕ
PWA-обёртка — итоговый файл ставится в систему как обычное приложение,
через установщик, со своим ярлыком, без браузера.

## Вариант А (проще всего): собрать автоматически на GitHub, без своего компьютера

В папке `.github/workflows/build.yml` уже лежит готовый рецепт для GitHub
Actions — он сам соберёт все три версии (Windows/macOS/Linux) на серверах
GitHub и выложит их в Releases. Вам не нужен Node.js, npm и три разных ОС.

1. Создайте новый репозиторий на GitHub (можно приватный) и залейте туда
   всю папку `electron-app` целиком (то, что вы сейчас скачали).
2. Отправьте тег версии:
   ```bash
   git tag v1.0.0
   git push origin v1.0.0
   ```
3. Откройте вкладку **Actions** в репозитории — сборка запустится сама
   (займёт 3-5 минут). Когда закончится, зайдите в **Releases** — там
   будут лежать `YTMetrics Setup.exe`, `YTMetrics.dmg`, `YTMetrics.AppImage`.
4. Ссылки на эти файлы (Releases → правой кнопкой на файл → "Копировать
   ссылку") вставьте мне — я подключу их на сайт в раздел «Скачать
   приложение», в кнопки Windows/macOS/Linux.

Готовую ссылку на самую свежую версию (не переписывать при каждом
релизе) можно взять по шаблону — просто подставьте свои `USER/REPO`:
```
https://github.com/USER/REPO/releases/latest/download/YTMetrics-Setup.exe
https://github.com/USER/REPO/releases/latest/download/YTMetrics.dmg
https://github.com/USER/REPO/releases/latest/download/YTMetrics.AppImage
```
Эта ссылка всегда указывает на файл из последнего релиза — при выходе
новой версии ничего на сайте менять не придётся.

## Вариант Б: собрать у себя на компьютере

## 1. Что нужно поставить один раз

- Node.js 18+ (https://nodejs.org)

## 2. Установка зависимостей

```bash
cd electron-app
npm install
```

## 3. Проверить, что всё открывается

```bash
npm start
```

Должно открыться окно с вашим сайтом. Если увидите пустой экран — проверьте
`APP_URL` в `main.js` (вдруг у сайта другой домен, отличный от github.io).

## 4. Собрать установщики

```bash
npm run dist:win     # .exe (NSIS-установщик) — собирается и на Windows, и на Mac/Linux (кросс-сборка)
npm run dist:mac     # .dmg — ТОЛЬКО на macOS (Apple не разрешает собирать mac-билды на других ОС)
npm run dist:linux   # .AppImage + .deb
npm run dist:all     # всё сразу (запускать на Mac, чтобы получить и mac, и остальные)
```

Готовые файлы появятся в папке `dist/`.

## 5. Про иконку

Я сгенерировал временную иконку-заглушку:
- `build/icon.png` — для Linux ✅ готова
- `build/icon.ico` — для Windows ✅ готова
- `build/icon.icns` — для macOS ❌ **отсутствует**, её нужно сделать отдельно,
  потому что формат `.icns` собирается только на macOS утилитой `iconutil`,
  либо любым онлайн-конвертером png → icns. Возьмите `build/icon_1024.png`
  (тоже сгенерирован) как исходник высокого разрешения.

Когда сделаете свой логотип — просто замените эти 3 файла в папке `build/`
(имена и размеры оставьте как есть) и пересоберите.

## 6. Подпись приложения (опционально, но важно)

Без подписи:
- **Windows** — SmartScreen покажет предупреждение "неизвестный издатель"
  (пользователь может нажать "всё равно запустить", но выглядит тревожно).
- **macOS** — Gatekeeper вообще не даст открыть без обхода через
  правый клик → "Открыть" (или `xattr -cr`).

Чтобы убрать эти предупреждения:
- Windows: нужен сертификат code signing (от ~$100-400/год у разных CA)
- macOS: нужен Apple Developer аккаунт ($99/год) + notarization

Это можно сделать позже, когда решите, что приложение готово к раздаче —
без подписи всё тоже будет работать, просто с предупреждением при первом запуске.

## 7. Автообновления (на будущее)

Если захотите, чтобы приложение само предлагало обновиться при выходе новой
версии — это делается через `electron-updater` + GitHub Releases (бесплатно,
не требует code signing для самого механизма обновления, хотя подпись всё
равно рекомендуется). Скажите — добавлю.
